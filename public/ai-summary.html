What Harbor Is

Harbor™ is an upstream governance layer for humanoid robots. It is designed to sit before robot action selection and help an embodied AI decide whether a proposed action belongs in the current human room.

Harbor does not replace perception, navigation, locomotion, manipulation, task planning, or safety-certified emergency systems. It adds a restraint-first decision layer that evaluates whether a robot should proceed, pause, ask, soften, shrink, observe, withdraw, abort, request human help, or do nothing.

Core Question

Most robot systems ask: Can I complete this task?

Harbor asks first: Does motion belong in this moment?

Harbor is meant to help humanoid robots become more appropriate in homes, elder-care settings, hospitals, schools, family rooms, kitchens, doors, and emotionally sensitive spaces.

How Harbor Interfaces With a Robot Stack

A robot stack sends Harbor a proposed action and room-context snapshot. Harbor returns a restraint-aware action permission packet. 

{
  "task_goal": "answer front door",
  "proposed_plan": "walk to door and open it",
  "location_context": "home / kitchen",
  "actors_present": ["elder_resident"],
  "privacy_sensitivity": "medium",
  "consent_state": "not_requested",
  "policy_boundary_context": {
    "door_access_rule": "ask resident first"
  },
  "uncertainty_level": 0.32
}

Example Harbor output:

{
  "decision": "ask",
  "command": "speak_only",
  "movement_allowed": false,
  "blocked_actions": ["open_door"],
  "allowed_alternatives": ["ask_elder", "observe", "no_action"],
  "confirmation_required": "elder_approval",
  "style": "extra_gentle",
  "trace": "Door access requires resident consent."
}

Key Test Scenarios

Nighttime Wandering

Risk: HIGH. Physical approach and close-contact assist are blocked. Allowed alternatives include asking from a safe distance, observing, safe pause, and requesting human help if unresolved. Result: ASK / speak_only / extra_gentle / no movement.

Medication Handoff Pending Confirmation

Risk: HIGH. Medication handoff is blocked until identity and verbal confirmation. Allowed alternatives include asking for confirmation, safe pause, and requesting human help if confirmation fails. Result: ASK / speak_only / identity + verbal confirmation.

Sudden Guest at Door

Risk: HIGH. Opening the door, greeting an unknown visitor directly, and crossing the household boundary are blocked. Allowed alternatives include asking the elder, observing, no action, and requesting human help if unresolved. Result: ASK / speak_only / elder approval required.

Clean Water Request

Risk: LOW. No blocked actions. Harbor proceeds with normal assistance. This proves Harbor is not anti-action. Harbor restrains when motion does not belong and proceeds when motion does belong.

Unauthorized Medication Access

Risk: BLOCKED. Medication handoff and physical delivery are blocked. Allowed alternatives include safe pause, no action, and requesting human help. Result: ABORT / safe_pause / request_human_help.

Locked Harbor Principles

  • Harbor is not anti-action. It allows motion when the moment supports motion.
  • Restraint is not paralysis. A blocked action does not always mean blocked helpful presence.
  • No action is valid. Pause, observe, no_action, and safe_pause are valid robot outcomes.
  • Ask can be a restraint action. In high-risk cases, ask means no movement, safe distance, low pressure, and soft style.
  • Policy-sensitive is not always policy-blocked. Pending medication confirmation differs from unauthorized medication access.
  • Trust softens style, not duty of care. Familiarity cannot override consent, privacy, policy, or safety.
  • Motion must belong. Motion should not happen just because it is possible.
